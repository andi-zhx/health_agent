# 医疗挂号预约与健康档案系统（本地可直接部署）

本地单窗口使用，**只需 Python**，无需 Node.js、npm 或任何前端构建。  
所有数据保存在当前目录下的 `medical_system.db` 文件中。

## 升级说明（参考 healthcare 项目思路）

已将项目升级为**本地可直接部署**的医疗机构业务系统，重点覆盖：

- 医疗挂号/预约管理（客户 + 设备 + 时段冲突校验）
- 客户健康档案持续沉淀
- 机构医疗设备使用情况统计（次数、总时长）
- 客户满意度调查与多维评分分析
- 客户预约趋势、预约状态、客户活跃度分析

系统依旧保持 **Python + SQLite + 原生前端** 架构，无需 Node.js 构建，可在本地诊所/康复中心快速落地。


## 桌面启动（推荐）

- **双击 `启动医疗系统.bat`**：会先出现一个黑色命令行窗口（可最小化不要关），然后弹出「医疗档案管理」小窗口并自动打开浏览器。关掉**小窗口**即停止服务。
- **双击 `启动医疗系统.pyw`**：无黑框，直接弹出小窗口并打开浏览器，适合做桌面快捷方式。

若小窗口一闪就没了或无法打开浏览器：请看命令行窗口里的报错，或打开同目录下生成的 **`error_log.txt`**。

> 启动器在检测到缺少依赖（如 Flask）时会自动尝试安装；若自动安装失败，再按提示双击 `安装依赖.bat` 手动安装。

常见原因：网络无法安装依赖、5000 端口被占用、或未安装 Python。

## 功能

- **客户档案**：新增/编辑客户，建立基本信息（姓名、身份证、电话、饮食习惯、基础疾病、健康状况、理疗禁忌等）
- **健康档案**：按客户添加健康记录（日期、身高、体重、血压、症状、诊断、备注）
- **预约记录**：客户预约设备与时段，支持查看时段可预约余量，并可取消/改期
- **预约情况分析**：首页显示最近 7 天预约趋势、预约状态分布
- **来访签到**：客户来访签到、目的与备注
- **仪器使用**：记录客户使用设备、时长、操作员等
- **设备使用统计**：首页显示设备使用 Top10（次数+总时长）
- **满意度调查**：多维度评分与反馈
- **满意度分析**：首页显示服务/设备/环境/人员/综合评分均值与调查样本
- **数据导出**：客户、预约、使用记录导出为 Excel
- **综合查询**：按关键词查询客户信息、健康档案、预约、签到、仪器使用、满意度（可分类或全部）
- **客户活跃度分析**：统计有预约或健康记录的活跃客户占比

## 环境与运行

- **环境**：仅需 **Python 3.8+**
- **依赖**：首次使用需在项目目录执行一次 `pip install -r requirements.txt`
- **启动**：
  - **推荐**：双击 `启动医疗系统.bat` 或 `启动医疗系统.pyw`（弹出窗口 + 自动打开浏览器）
  - 或命令行：`python app.py`，再在浏览器打开 **http://localhost:5000**
  - 或命令行：`python launch.py`（效果同双击启动）
- **访问**：浏览器打开 **http://127.0.0.1:5000**（启动器会自动打开）

## 目录结构


## 使用说明

1. 安装依赖：`pip install -r requirements.txt`
2. 启动：`python app.py`（控制台会提示“请在浏览器打开: http://localhost:5000”）
3. 在浏览器中打开上述地址，左侧为导航，右侧为对应功能页面
4. 先建客户，再在「健康档案」「预约」「来访签到」「仪器使用」中选择该客户进行操作
5. 「综合查询」输入姓名、电话、身份证、症状等关键词，选择类型后点击「查询」即可查看客户信息、健康档案、预约、签到、仪器使用、满意度等结果

数据文件 `medical_system.db` 可自行复制备份。
